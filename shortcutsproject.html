iOS Shortcuts Redesign


Redesigning the iOS Shortcuts app to eliminate design inconsistencies and reduce friction in the user's workflow. Aimed to provide more user clarity and lower the learning curve for beginner users.

UX Designer

Grad School Project

2 months (2024)

Figma

Project Overview


Context

Final project for our HCID 570 class, which explored design patterns & contexts normally found in everyday world. My team chose to revamp the iOS Shortcuts app, citing its unintuitive user experience and lack of in-app support as potential problems to address.

Brief

Simplify the process of creating shortcuts, shorten the learning curve, and improve usability.

Design Approach

Target Users

  • Apple users
  • Beginners who are unfamiliar with Shortcuts

Research Methods

  • Heuristics Evaluation
  • User Surveys
  • Contextual Inquiry

Design Goals

01

Improve onboarding experience

02

Ensure consistency in design elements and functionalities

03

Simplify action selection workflow

04

Streamline creating, editing, and testing shortcut processes

05

Provide in-app support for additional helpful information

Heuristics Evaluation

01

Visibility of System Status

Users receive no clear feedback on performed actions while editing shortcuts, especially incorrect actions.

02

Error Recognition & Recovery

The app returns uninformative error messages that provide no actionable insights.

03

System & Real World Match

App language features many technical jargons which adds to user confusion.

04

Flexibility & Efficiency of Use

Users are immediately introduced to an overwhelming list of all shortcut actions, which significantly hinders beginners' learning process.

05

User Control & Freedom

"Undo" and "Redo" functions are not prominently presented, discouraging users from experimenting for fear of making irreversible mistakes.

06

Error Prevention

Lacking adequate guardrails and user suggestions, the app fails to prevent simple mistakes such as choosing incompatible actions or leaving fields blank.

07

Consistency & Standards

Some design elements aren't aligned with Apple's design system guidelines. Elements that do look the same don't function the same way.

08

Recognition Rather than Recall

Icons lack text labels, and the app doesn't provide helpful onscreen cues to aid shortcut creation.

09

Help & Documentation

Minimal in-app support and guidance adds more barriers for the users to learn how to use the app.

User Surveys

Contextual Inquiry

Methodology

  • Observation: Users were observed while attempting to create a shortcut
  • Interviews: Conducted post observation interviews to gather user feedback and insights

Key Findings

01

Users heavily relied on external sources to educate themselves on the app's functionalities.

02

Users easily misinterpreted or outright missed certain functionalities while trying to create and edit the shortcut.

03

Users had difficulty navigating the app and understanding where they were supposed to go.
